# Contributing to Self-Refine Skill

Thank you for your interest in improving this skill!

## How to Contribute

### Reporting Issues

- Open an [Issue](https://github.com/Thomaszhou22/self-refine-skill/issues)
- Include: platform, expected behavior, actual behavior, steps to reproduce

### Suggesting Improvements

- Open an Issue with the label `enhancement`
- Describe the current behavior and your proposed change
- If possible, reference academic research supporting the change

### Submitting Changes

1. Fork the repository
2. Create a feature branch (`git checkout -b feature/your-improvement`)
3. Make your changes
4. If changing SKILL.md, verify the change improves at least one reflection dimension
5. Submit a Pull Request

### What We're Looking For

- **New reflection dimensions** backed by research
- **Platform-specific optimizations** (better auto-detection, new platforms)
- **Better inline templates** for constrained environments
- **Benchmark improvements** (more test cases, better scoring methodology)
- **Translations** of SKILL.md instructions for non-English users

### What We Won't Accept

- Changes that remove academic references without replacement
- Generic "write better" rules (that's not what this skill is about)
- Breaking changes to the 4-level depth system without strong justification

## Style Guide

- All documentation in English
- Academic references must include: authors, year, title, arXiv/DOI link
- Code examples should be runnable without modification
- Tables use consistent alignment (center for ratings, left for text)
